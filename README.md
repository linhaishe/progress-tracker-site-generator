# progress-tracker-site-generator

Generate a read-only static progress dashboard from standard project tracker files.

This package is published as `@freeasaka/progress-tracker-site-generator`. It reads or creates:

- `context/progress-tracker.md`
- `context/milestone-tracker.md`

Then it generates a self-contained HTML page that can be opened directly in a browser. The page is designed as a handoff cockpit for developers and AI agents: current focus, next workflow, milestones, blockers, risks, assumptions, and verification status.

## Usage

After the package is published to npm:

```bash
npx @freeasaka/progress-tracker-site-generator init
# or
pnpm dlx @freeasaka/progress-tracker-site-generator init
```

Before npm publishing, install directly from GitHub:

```bash
npx github:freeasaka/progress-tracker-site-generator init
# or
pnpm dlx github:freeasaka/progress-tracker-site-generator init
```

By default, the command writes:

- `docs/progress-tracker.html` when the target project already has a `docs/` directory
- `progress-tracker.html` at the project root otherwise

Useful options:

```bash
npx @freeasaka/progress-tracker-site-generator init --dry-run
npx @freeasaka/progress-tracker-site-generator init --force
npx @freeasaka/progress-tracker-site-generator init --root .
npx @freeasaka/progress-tracker-site-generator init --output docs/progress-tracker.html
npx @freeasaka/progress-tracker-site-generator init --watch --output docs/progress-tracker.html
npx @freeasaka/progress-tracker-site-generator init --ai
```

With pnpm, replace `npx` with `pnpm dlx`:

```bash
pnpm dlx @freeasaka/progress-tracker-site-generator init --watch --output docs/progress-tracker.html
```

AI initialization is opt-in. By default, `init` stays offline and uses templates/rules only. With `--ai`, the command sends matching target-project docs to OpenAI and asks for a synthesized `context/progress-tracker.md` instead of copying source documents into it:

```bash
OPENAI_API_KEY=... pnpm dlx @freeasaka/progress-tracker-site-generator init --ai
```

You can override the model with `OPENAI_MODEL`; the default is `gpt-4.1-mini`.

If your terminal needs a local proxy to reach OpenAI, pass it with standard proxy environment variables:

```bash
HTTPS_PROXY=http://127.0.0.1:7890 OPENAI_API_KEY=... pnpm dlx @freeasaka/progress-tracker-site-generator init --ai
```

Moonshot/Kimi is also supported through its OpenAI-compatible chat completions API:

```bash
AI_PROVIDER=moonshot \
MOONSHOT_API_KEY=... \
AI_BASE_URL=https://api.moonshot.cn/v1 \
AI_MODEL=kimi-k2.6 \
HTTPS_PROXY=http://127.0.0.1:7890 \
pnpm dlx @freeasaka/progress-tracker-site-generator init --ai
```

Gemini is supported through Google's OpenAI-compatible chat completions API:

```bash
AI_PROVIDER=gemini \
GEMINI_API_KEY=... \
AI_MODEL=gemini-2.5-flash \
HTTPS_PROXY=http://127.0.0.1:7890 \
pnpm dlx @freeasaka/progress-tracker-site-generator init --ai
```

`--watch` runs an initial generation, then keeps the process open and regenerates the HTML whenever these files change:

- `context/progress-tracker.md`
- `context/milestone-tracker.md`
- `context/design.md`

## Behavior

- Existing tracker markdown files are preserved.
- Missing tracker files are created from generic templates.
- When `context/progress-tracker.md` is missing, `init` also scans `docs/` for related markdown files whose paths mention PRD, product requirements, implementation, stage plan, roadmap, or milestones.
- In default offline mode, matching file contents are appended to the initial progress tracker under `Project Docs Context`.
- In `--ai` mode, matching target-project docs are used as source material for a synthesized progress tracker. Packaged `templates/docs/` files are used only as reference material for structure and expectations.
- If `--ai` cannot run, for example because `OPENAI_API_KEY` is missing, `init` falls back to default offline mode and reports an `AI skipped` note.
- Existing non-generated HTML files are not overwritten.
- `--force` only overwrites HTML previously generated by this package.
- `--watch` regenerates existing generated HTML automatically and behaves like `--force` for the HTML output.
- The generated page is static and read-only.
- The browser page can paste or load updated markdown to refresh the visible dashboard, but it never writes files.
